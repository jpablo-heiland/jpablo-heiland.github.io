1 Introducción

El presente informe detalla el proceso de Zonificación de un lote de producción mediante el análisis de tres mapas de rendimiento de soja, una imagen NDVI del cultivo de soja y la altimetría del lote.

El rendimiento es considerado una variable síntesis; es decir, el resultado final donde convergen miles de factores que interactúan entre sí. Si bien por sí solo no explica las causas de la variabilidad, constituye a menudo la fuente de información más accesible y completa para entender qué sucedió en el lote.

El flujo de trabajo propuesto se dividió en:

I. Carga de Datos: Se incorporaron los mapas de rendimiento correspondientes a las campañas 2009, 2011 y 2013, junto con el mapa de NDVI. Los datos de altimetría se extrajeron directamente a partir de los registros del mapa de rendimiento 2011.

II. Creación de Grilla: En la cosecha la maquina recolecta datos de campo que están mas cercas en la línea que entre líneas. Esto provoca cierta estructura espacial ficticia de los datos. Para evitar esto se procedió a generar una discretización continua del espacio mediante una cuadrícula regular de polígonos. En esta se unieron los datos de rendimiento, altimetría y NDVI.

III. Cálculo Indice de Productividad: Para que los datos de diferentes campañas fueran comparables entre sí, los valores absolutos de rendimiento se transformaron en un Índice de Productividad (IP).

IV. Selección de Variables mediante Análisis de Regresión: Se realizó un análisis de regresión lineal para evaluar la capacidad de las variables de explicar la variabilidad observada en el IP. Este paso es fundamental para asegurar que las variables seleccionadas para el PCA y el posterior Fuzzy k-means posean una relevancia comprobada.

V. Análisis de Componentes Principales Espaciales (MULTISPATI-PCA): Esta técnica permite reducir la dimensionalidad de un conjunto de datos, extrayendo “componentes” que explican la mayor parte de la variabilidad total. A diferencia del PCA clásico, el MULTISPATI-PCA incorpora la autocorrelación espacial mediante el Índice de Moran, asegurando que los datos sean analizados considerando la influencia de sus vecinos directos.

VI. Clasificación Multivariada (Fuzzy K-Means clustering): Una vez procesados los componentes principales, se aplica este algoritmo de agrupamiento “difuso”. A diferencia de los métodos deterministas, el sistema Fuzzy permite que los puntos tengan grados de pertenencia a diferentes zonas, lo que resulta ideal para la agricultura donde las transiciones en el suelo no son líneas tajantes sino gradientes.

VII. Creación de zonas de manejo: A partir de los resultados de Fuzzy K-Means cada zona de manejo se representó en polígonos.

3 Carga de Datos

La lectura de datos se realizó por medio de la función st_read() del paquete sf. Esta función lee archivos geoespaciales (Shapefile, GeoPackage, GeoJSON, KML, etc.) y los convierte en objetos Simple Feature (sf).

Un objeto de clase sf es, esencialmente, un data.frame que tiene una columna especial llamada geometry`, la cual almacena la representación espacial (puntos, líneas o polígonos) que corresponde a cada fila.

4 Cuadrícula

En este punto se procedió a la creación de la Cuadrícula. Posteriormente se unieron a ella los datos de rendimiento 2009, 2011, 2013, altimetría y elevación.

4.1 Cargar la capa del Lote

lote <- st_read("data/Lote14.shp", quiet = TRUE)

4.2 Crear grilla

Se usó la función st_make_grid() de sf para crear la grilla. Por medio del parámetro cellsize se determina que cada polígono de la grilla tiene una dimensión de 10 x 10 m.

grilla <- st_make_grid(lote, cellsize = c(10, 10), what = "polygons", square = TRUE)

4.3 Convertir la grilla a objeto Simple Feature (sf)

grilla_sf <- st_as_sf(grilla)

Al convertir la grilla a sf cada fila queda vinculada permanentemente a un polígono geométrico que representa una celda en el lote. El paquete sf es el estándar para el manejo de datos vectoriales. Al tener los datos en este formato se puede hacer la Unión Espacial de los 3 mapas de rindes a la cuadrícula.

4.4 Crear un ID a la grilla

grilla_sf <- grilla_sf %>% mutate(id_celda = row_number())

4.5 Unir Rendimientos a la Grilla

Para integrar la información de los 3 mapas de rendimiento, altimetría y NDVI a la grilla, se aplicó el siguiente flujo:

  • Se realizó una unión espacial por medio de la función st_join() con el predicado st_intersects para identificar qué puntos de rendimiento caen dentro de cada celda de la grilla.

  • Dado que una sola celda de la grilla suele contener múltiples puntos registrados, se procedió a agrupar los datos en función del id_celda, mediante group_by().

  • Finalmente, se calculó el valor medio de rendimiento para cada unidad de la grilla (RTO_09 = mean) con summarise().

Los datos de Altimetría proceden del set de datos de Soja 2011, tal como se observa en la Tabla 2 (Elev11).

  • Unir Rendimiento 2009
  • Unir Rendimiento 2011 y Altimetría
  • Unir Rendimiento 2013
  • Unir NDVI
grilla_09 <- st_join( 
x = grilla_sf, 
y = soja2009 %>% dplyr::select(Rto09), 
  join = st_intersects 
)

grilla_09 <- grilla_09 %>% 
  group_by(id_celda) %>%          
  summarise( 
    n_puntos = n(),               
    RTO_09 = mean(Rto09, na.rm = TRUE)  
  )
grilla_11 <- st_join( 
  x = grilla_sf,
  y = soja2011 %>% dplyr::select(RTO11, Elev11),

) 

grilla_11 <- grilla_11 %>% 
  group_by(id_celda) %>%        
  summarise( 
    n_puntos = n(),              
    RTO_11   = mean(RTO11, na.rm = TRUE),

  ) 
grilla_13 <- st_join( 
  x = grilla_sf,

) 
 
grilla_13 <- grilla_13 %>%

4.6 Unir las 4 grillas

Se unieron las capas de información en un único set de datos (grilla_rto). El procedimiento se realizó mediante uniones relacionales por la izquierda (left_join) tomando como base la grilla_09. Con el propósito evitar conflictos de redundancia, se removió la columna geometry de las capas secundarias (grilla_11, grilla_13 y grilla_NDVI) mediante la función st_drop_geometry(), logrando así la unión de todos los datos a través del identificador común (id_celda`.

grilla_rto <- grilla_09 %>% 
  left_join(st_drop_geometry(grilla_11), by = "id_celda") %>% 
  left_join(st_drop_geometry(grilla_13), by = "id_celda") %>%
  left_join(st_drop_geometry(grilla_NDVI), by = "id_celda")

Eliminar los polígonos de la grilla con valores NA, que complican los resultados posteriores:

grilla_rto <- grilla_rto %>% 
  filter( 
    !is.na(RTO_13), 
    !is.na(RTO_11), 
    !is.na(RTO_09),
    !is.na(NDVI25)
    )

5 IP

El Índice de Productividad (IP) es una medida de rendimiento relativo. Se calcula dividiendo el rendimiento de cada punto individual por el rendimiento promedio de todo el lote.

Permite comparar zonas de un lote con otros lotes que tienen potenciales de rendimiento diferentes.

Interpretación:

  • IP > 1: La unidad rinde por encima del promedio. Un IP de 1.15 significa que rinde un 15% más que la media.
  • IP = 1: La unidad rinde lo mismo que el promedio.
  • IP < 1: La nidad rinde por debajo del promedio. Un IP de 0.85 significa que rinde un 15% menos que la media.

Cálculo:

grilla_IP <- grilla_rto %>% 
  mutate( 
    IP = ( 
      (RTO_09 / mean(RTO_09, na.rm = TRUE)) + 
        (RTO_11 / mean(RTO_11, na.rm = TRUE)) + 
        (RTO_13 / mean(RTO_13, na.rm = TRUE)) 
    ) / 3 
  )

5.1 Graficar el IP

La visualización del IP se estructuró mediante una discretización en cuantiles (4), para ello, primero se deben crear estos cuantiles:

breaks <- quantile(grilla_IP$IP, probs = seq(0, 1, 0.25), na.rm = TRUE)

8 Multispati-PCA

Para poder ejecutar el MULTISPATI-PCA, es un requisito estructurar el espacio, definiendo cómo interactúan las celdas de la grilla entre sí. Esto se construye en dos etapas sucesivas:

1. Identificación de Vecinos: El primer paso es crear una lista de vecindad basada en distancias euclidianas. A partir de las coordenadas de los centroides de la grilla, la función dnearneigh (paquete spdep) evalúa qué puntos se encuentran dentro de un radio de distancia específico, en este caso de 35 metros. Todas las celdas que caigan dentro de esa distancia umbral son catalogadas formalmente como “vecinas”.

2. Ponderación de Influencia Espacial: Una vez definido quiénes son los vecinos, es necesario determinar cuánta influencia ejercen. La función nb2listw transforma la lista de vecindades en una matriz de pesos espaciales.

9.5 Calcular Índice Resumen

Para determinar el número óptimo de zonas de manejo (\(k = 2, 3, 4\)), se calculó un índice resumen para cada clasificación, esto unifica el comportamiento de los cuatro índices de validación difusa (\(XB\), \(PC\), \(PE\), \(FS\)). El procedimiento siguió los siguientes pasos:

El primer paso fue extraer las respuestas numéricas de cada índice para las diferentes clasificaciones de clústeres analizadas a partir del objeto indices_todos, convirtiéndolas en vectores continuos mediante as.numeric():

xb <- as.numeric(indices_todos[1, 2:4])
pc <- as.numeric(indices_todos[2, 2:4])
pe <- as.numeric(indices_todos[3, 2:4])
fs <- as.numeric(indices_todos[4, 2:4])

Cada vector contiene tres valores, uno por número de clases, representando el comportamiento de ese índice a lo largo de las particiones evaluadas.

Dado que los índices tienen distintas escalas y unidades, no son comparables directamente. Se aplica una normalización Min-Max que proyecta cada vector al intervalo [0,1][0, 1] [0,1], reorientada de modo que el valor 0 siempre corresponda al comportamiento ideal de cada índice

Para XB, PE y FS, donde el criterio de optimalidad es minimizar, la normalización se realiza de la siguiente manera:

\[I_{\text{norm}} = \frac{I - I_{\min}}{I_{\max} - I_{\min}}\]

XieBeniN  <- (xb - min(xb)) / (max(xb) - min(xb))
EntrPartN <- (pe - min(pe)) / (max(pe) - min(pe))
FukSugN   <- (fs - min(fs)) / (max(fs) - min(fs))

Para PC, donde el criterio es maximizar, se invierte la fórmula de modo que el máximo quede en 0 y el mínimo en 1:

\[I_{\text{norm}} = \frac{I_{\max} - I}{I_{\max} - I_{\min}}\]

CoefPartN <- (max(pc) - pc) / (max(pc) - min(pc))

Tras esta transformación, los cuatro vectores normalizados comparten la misma semántica: valores más cercanos a 0 indican mejor desempeño.

Los cuatro vectores normalizados se consolidan en un data.frame de dimensión 4 x 3 (cuatro índices, tres particiones), cuyos valores se elevan al cuadrado como paso intermedio del cálculo de la distancia euclídea:

indicesN <- data.frame(rbind(XieBeniN, CoefPartN, EntrPartN, FukSugN))
indicesN_2 <- indicesN^2

Finalmente, se calcula el Índice Resumen (IR) para cada nivel de partición (\(k\)) mediante la aplicación de la norma Euclídea (\(L_2\)). Geométricamente, el algoritmo proyecta las puntuaciones de los cuatro índices estandarizados como coordenadas dentro de un espacio tetradimensional (\(\mathbb{R}^4\)).

El “Punto Ideal” está definido por el vector \((0, 0, 0, 0)\), donde todos los índices alcanzarían simultáneamente su mejor rendimiento posible. El Índice Resumen para cada clase \(k\) se calcula formalmente como:

\[IR_k = \sqrt{\sum_{i=1}^{m} (I_{i,k,\text{norm}})^2}\]

indices2N <- sqrt(sum(indicesN_2[, 1])) 
indices3N <- sqrt(sum(indicesN_2[, 2])) 
indices4N <- sqrt(sum(indicesN_2[, 3]))

10.1 Interpolación IDW

Se aisla únicamente la variable a interpolar (X3clases) junto con la geometría de puntos:

datos_3Clases <- base_am %>% 
  select(X3clases, geometry)

Cambiar el nombre de las columnas por un nombre genérico que simplifica esa referencia:

colnames(datos_3Clases) <- c("Clase", "geometry")

Convertir a objeto sp, La conversión es obligatoria porque gstat no acepta objetos sf directamente como argumento locations. El objeto resultante es un SpatialPointsDataFrame:

datos_sp <- as(datos_3Clases, "Spatial")

Crear grilla. Este es el destino de la interpolación: una grilla regular de celdas de 10 × 10 m que cubre la extensión del lote. Cada celda recibirá un valor interpolado de X3clases. El tamaño de celda determina la resolución espacial del resultado final (celdas más pequeñas producen polígonos más detallados pero mayor costo computacional).

El objeto pasa por tres estados: sfc (geometría pura de st_make_grid) → sf (para agregar el campo id_celda) → sp (requerido por gstat).

grilla_2 <- st_make_grid(datos_sp, cellsize = c(10, 10), what = "polygons", square = TRUE)

grilla_sf <- st_as_sf(grilla_2) 

grilla_sf <- grilla_sf %>% mutate(id_celda = row_number())

grilla_sp <- as(grilla_sf, "Spatial")

Interpolación IDW.

idw_model <- gstat::idw(formula = Clase ~ 1, locations = datos_sp, newdata = grilla_sp, idp = 2) 
[inverse distance weighted interpolation]

El resultado es un SpatialPixelsDataFrame con una columna var1.pred que contiene el valor interpolado en cada celda —un número continuo entre 1 y 3.

Se extrae el resultado a un data.frame con columnas x, y y var1.pred, y se reconstruye como un ráster con terra::rast(). El argumento type = “xyz” indica que las dos primeras columnas son coordenadas y la tercera es el valor de celda. Este paso es necesario porque las operaciones siguientes (reclasificación y vectorización) pertenecen al ecosistema terra, no a gstat.

idw_df <- as.data.frame(idw_model) 
r_pred <- terra::rast(idw_df[, c("x", "y", "var1.pred")], type = "xyz")

Reclasificar: La IDW es un interpolador de valores continuos. A los input de puntos con valores 1, 2 y 3 los trata como si fueran una variable cuantitativa cualquiera. El resultado es una superficie donde cada celda puede tomar cualquier valor real dentro del rango observado: una celda ubicada en la transición entre la zona 1 y la zona 2 puede tener un valor interpolado de 1.3, 1.7 o 2.1, dependiendo de qué tan cerca esté de cada grupo de puntos. Por eso es que se realiza un reclasificado.

La superficie continua se convierte de vuelta a clases enteras mediante umbrales equidistantes entre los valores de clase (1, 2, 3): el punto medio entre clase 1 y clase 2 es 1.5, y entre clase 2 y clase 3 es 2.5. Cualquier celda con valor interpolado por debajo de 1.5 queda asignada a la Clase 1, entre 1.5 y 2.5 a la Clase 2, y por encima de 2.5 a la Clase 3. La conversión con as.factor() explicita que el ráster resultante es categórico.

rcl <- matrix(c(-Inf, 1.5, 1, 
                1.5, 2.5, 2, 
                2.5, Inf, 3), 
              ncol = 3, byrow = TRUE) 

reclasificado <- classify(r_pred, rcl = rcl) %>% as.factor()

Vectorización y conversión final: as.polygons() con dissolve = TRUE agrupa todas las celdas contiguas de la misma clase y las fusiona en un único polígono por zona, eliminando los bordes internos de la grilla. El resultado es un SpatVector con tres campos (uno por zona). La conversión a sf y la asignación del CRS (EPSG:32720, WGS 84 / UTM zona 20S) devuelven el objeto estándar para realizar su gráfico:

# Introducción

El presente informe detalla el proceso de **Zonificación** de un lote de producción mediante el análisis de tres mapas de rendimiento de soja, una imagen NDVI del cultivo de soja y la altimetría del lote.

El **rendimiento** es considerado una variable síntesis; es decir, el resultado final donde convergen miles de factores que interactúan entre sí. Si bien por sí solo no explica las causas de la variabilidad, constituye a menudo la fuente de información más accesible y completa para entender qué sucedió en el lote.

El flujo de trabajo propuesto se dividió en:

**I. Carga de Datos**: Se incorporaron los mapas de rendimiento correspondientes a las campañas 2009, 2011 y 2013, junto con el mapa de NDVI. Los datos de altimetría se extrajeron directamente a partir de los registros del mapa de rendimiento 2011.

**II. Creación de Grilla**: En la cosecha la maquina recolecta datos de campo que están mas cercas en la línea que entre líneas. Esto provoca cierta estructura espacial ficticia de los datos. Para evitar esto se procedió a generar una discretización continua del espacio mediante una cuadrícula regular de polígonos. En esta se unieron los datos de rendimiento, altimetría y NDVI.

**III. Cálculo Indice de Productividad**: Para que los datos de diferentes campañas fueran comparables entre sí, los valores absolutos de rendimiento se transformaron en un Índice de Productividad (IP).

**IV. Selección de Variables mediante Análisis de Regresión**: Se realizó un análisis de regresión lineal para evaluar la capacidad de las variables de explicar la variabilidad observada en el IP. Este paso es fundamental para asegurar que las variables seleccionadas para el PCA y el posterior Fuzzy k-means posean una relevancia comprobada.

**V. Análisis de Componentes Principales Espaciales (MULTISPATI-PCA)**: Esta técnica permite reducir la dimensionalidad de un conjunto de datos, extrayendo "componentes" que explican la mayor parte de la variabilidad total. A diferencia del PCA clásico, el MULTISPATI-PCA incorpora la autocorrelación espacial mediante el Índice de Moran, asegurando que los datos sean analizados considerando la influencia de sus vecinos directos.

**VI. Clasificación Multivariada (Fuzzy K-Means clustering)**: Una vez procesados los componentes principales, se aplica este algoritmo de agrupamiento "difuso". A diferencia de los métodos deterministas, el sistema Fuzzy permite que los puntos tengan grados de pertenencia a diferentes zonas, lo que resulta ideal para la agricultura donde las transiciones en el suelo no son líneas tajantes sino gradientes.

**VII. Creación de zonas de manejo**: A partir de los resultados de Fuzzy K-Means cada zona de manejo se representó en polígonos.

# IP {#sec-ip}

El **Índice de Productividad (IP)** es una medida de rendimiento relativo. Se calcula dividiendo el rendimiento de cada punto individual por el rendimiento promedio de todo el lote.

Permite comparar zonas de un lote con otros lotes que tienen potenciales de rendimiento diferentes.

Interpretación:

-   IP \> 1: La unidad rinde por encima del promedio. Un IP de 1.15 significa que rinde un 15% más que la media.
-   IP = 1: La unidad rinde lo mismo que el promedio.
-   IP \< 1: La nidad rinde por debajo del promedio. Un IP de 0.85 significa que rinde un 15% menos que la media.

Cálculo:

```{r}
grilla_IP <- grilla_rto %>% 
  mutate( 
    IP = ( 
      (RTO_09 / mean(RTO_09, na.rm = TRUE)) + 
        (RTO_11 / mean(RTO_11, na.rm = TRUE)) + 
        (RTO_13 / mean(RTO_13, na.rm = TRUE))

  )
```

# Multispati-PCA

Para poder ejecutar el MULTISPATI-PCA, es un requisito estructurar el espacio, definiendo cómo interactúan las celdas de la grilla entre sí. Esto se construye en dos etapas sucesivas:

**1. Identificación de Vecinos**: El primer paso es crear una lista de vecindad basada en distancias euclidianas. A partir de las coordenadas de los centroides de la grilla, la función `dnearneigh` (paquete [spdep]{.func-tecnica}) evalúa qué puntos se encuentran dentro de un radio de distancia específico, en este caso de 35 metros. Todas las celdas que caigan dentro de esa distancia umbral son catalogadas formalmente como "vecinas".

**2. Ponderación de Influencia Espacial**: Una vez definido quiénes son los vecinos, es necesario determinar cuánta influencia ejercen. La función `nb2listw` transforma la lista de vecindades en una matriz de pesos espaciales.

## Identificación de Vecinos

En primer lugar, se extraen las coordenadas:

```{r}
cord2 <- coordinates(df_centroides[, 1:2]) 
```

> [cord2]{.func-tecnica} es una matriz de coordenadas (dos columnas: X e Y), que se usará para definir los vecindarios espaciales.

Crear la matriz de vecinos. Como se mencionó, se indica que dos puntos son vecinos si están entre 0 y 35 unidades de distancia:
```{r}
vecindarios <- dnearneigh(cord2, 0, 35)
```

## Matriz de Pesos Espaciales

```{r}
pesos_esp <- nb2listw(vecindarios, 
                     style = "W", 
                     zero.policy = T)
```

## Multispati-PCA

Se usó la función `multispati()` del paquete [adespatial]{.func-tecnica}. Su propósito es incorporar la autocorrelación espacial (es decir, la semejanza entre puntos cercanos) dentro de un análisis de componentes principales.

```{r}
pca_esp <- adespatial::multispati(pca, pesos_esp, scannf = F, nfposi = 5)
```

## Resultado

Se guarda un objeto “multispati” llamado [pca_esp]{.func-tecnica}, con varios elementos:

-   [eig]{.func-tecnica}: Son los Autovalores espaciales, explica cuánta varianza espacial explica cada componente.
-   [var]{.func-tecnica}: Varianza explicada por cada componente del PCA clásico, representa cuánta variabilidad original capta cada eje
-   [cum]{.func-tecnica}: Varianza acumulada
-   [ratio]{.func-tecnica}: Porcentaje acumulado (normalizado entre 0 y 1)\
-   [moran]{.func-tecnica}: índice de Moran de cada componente

## Calcular Índice Resumen

Para determinar el número óptimo de zonas de manejo ($k = 2, 3, 4$), se calculó un **índice resumen** para cada clasificación, esto unifica el comportamiento de los cuatro índices de validación difusa ($XB$, $PC$, $PE$, $FS$). El procedimiento siguió los siguientes pasos:

El primer paso fue extraer las respuestas numéricas de cada índice para las diferentes clasificaciones de clústeres analizadas a partir del objeto [indices_todos]{.func-tecnica}, convirtiéndolas en vectores continuos mediante `as.numeric()`:

```{r}
xb <- as.numeric(indices_todos[1, 2:4])
pc <- as.numeric(indices_todos[2, 2:4])
pe <- as.numeric(indices_todos[3, 2:4])
fs <- as.numeric(indices_todos[4, 2:4])
```

> Cada vector contiene tres valores, uno por número de clases, representando el comportamiento de ese índice a lo largo de las particiones evaluadas.

Dado que los índices tienen distintas escalas y unidades, no son comparables directamente. Se aplica una normalización Min-Max que proyecta cada vector al intervalo \[0,1\]\[0, 1\] \[0,1\], reorientada de modo que el valor 0 siempre corresponda al comportamiento ideal de cada índice

Para *XB*, *PE* y *FS*, donde el criterio de optimalidad es minimizar, la normalización se realiza de la siguiente manera:

$$I_{\text{norm}} = \frac{I - I_{\min}}{I_{\max} - I_{\min}}$$

```{r}
XieBeniN  <- (xb - min(xb)) / (max(xb) - min(xb))
EntrPartN <- (pe - min(pe)) / (max(pe) - min(pe))
FukSugN   <- (fs - min(fs)) / (max(fs) - min(fs))
```

Para *PC*, donde el criterio es maximizar, se invierte la fórmula de modo que el máximo quede en 0 y el mínimo en 1:

$$I_{\text{norm}} = \frac{I_{\max} - I}{I_{\max} - I_{\min}}$$

```{r}
CoefPartN <- (max(pc) - pc) / (max(pc) - min(pc))
```

Tras esta transformación, los cuatro vectores normalizados comparten la misma semántica: valores más cercanos a 0 indican mejor desempeño.

Los cuatro vectores normalizados se consolidan en un [data.frame]{.func-tecnica} de dimensión 4 x 3 (cuatro índices, tres particiones), cuyos valores se elevan al cuadrado como paso intermedio del cálculo de la distancia euclídea:

```{r}
indicesN <- data.frame(rbind(XieBeniN, CoefPartN, EntrPartN, FukSugN))
indicesN_2 <- indicesN^2
```

Finalmente, se calcula el **Índice Resumen (IR)** para cada nivel de partición ($k$) mediante la aplicación de la norma Euclídea ($L_2$). Geométricamente, el algoritmo proyecta las puntuaciones de los cuatro índices estandarizados como coordenadas dentro de un espacio tetradimensional ($\mathbb{R}^4$).

El "Punto Ideal" está definido por el vector $(0, 0, 0, 0)$, donde todos los índices alcanzarían simultáneamente su mejor rendimiento posible. El Índice Resumen para cada clase $k$ se calcula formalmente como:

$$IR_k = \sqrt{\sum_{i=1}^{m} (I_{i,k,\text{norm}})^2}$$

```{r}
indices2N <- sqrt(sum(indicesN_2[, 1])) 
indices3N <- sqrt(sum(indicesN_2[, 2])) 
indices4N <- sqrt(sum(indicesN_2[, 3])) 
```

Crear grilla. Este es el destino de la interpolación: una grilla regular de celdas de 10 × 10 m que cubre la extensión del lote. Cada celda recibirá un valor interpolado de [X3clases]{.func-tecnica}. El tamaño de celda determina la resolución espacial del resultado final (celdas más pequeñas producen polígonos más detallados pero mayor costo computacional).

El objeto pasa por tres estados: [sfc]{.func-tecnica} (geometría pura de `st_make_grid`) → [sf]{.func-tecnica} (para agregar el campo [id_celda]{.func-tecnica}) → [sp]{.func-tecnica} (requerido por [gstat]{.func-tecnica}).

```{r}
grilla_2 <- st_make_grid(datos_sp, cellsize = c(10, 10), what = "polygons", square = TRUE)

grilla_sf <- st_as_sf(grilla_2) 

grilla_sf <- grilla_sf %>% mutate(id_celda = row_number())

grilla_sp <- as(grilla_sf, "Spatial") 
```

Interpolación IDW.

```{r}
idw_model <- gstat::idw(formula = Clase ~ 1, locations = datos_sp, newdata = grilla_sp, idp = 2) 
```

El resultado es un [SpatialPixelsDataFrame]{.func-tecnica} con una columna [var1.pred]{.func-tecnica} que contiene el valor interpolado en cada celda —un número continuo entre 1 y 3.

Se extrae el resultado a un [data.frame]{.func-tecnica} con columnas [x]{.func-tecnica}, [y]{.func-tecnica} y [var1.pred]{.func-tecnica}, y se reconstruye como un ráster con `terra::rast()`. El argumento [type = "xyz"]{.func-tecnica} indica que las dos primeras columnas son coordenadas y la tercera es el valor de celda. Este paso es necesario porque las operaciones siguientes (reclasificación y vectorización) pertenecen al ecosistema [terra]{.func-tecnica}, no a [gstat]{.func-tecnica}.

```{r}
idw_df <- as.data.frame(idw_model) 
r_pred <- terra::rast(idw_df[, c("x", "y", "var1.pred")], type = "xyz")
```

Reclasificar: La IDW es un interpolador de valores continuos. A los input de puntos con valores 1, 2 y 3 los trata como si fueran una variable cuantitativa cualquiera. El resultado es una superficie donde cada celda puede tomar cualquier valor real dentro del rango observado: una celda ubicada en la transición entre la zona 1 y la zona 2 puede tener un valor interpolado de 1.3, 1.7 o 2.1, dependiendo de qué tan cerca esté de cada grupo de puntos. Por eso es que se realiza un **reclasificado**.

La superficie continua se convierte de vuelta a clases enteras mediante umbrales equidistantes entre los valores de clase (1, 2, 3): el punto medio entre clase 1 y clase 2 es 1.5, y entre clase 2 y clase 3 es 2.5. Cualquier celda con valor interpolado por debajo de 1.5 queda asignada a la Clase 1, entre 1.5 y 2.5 a la Clase 2, y por encima de 2.5 a la Clase 3. La conversión con `as.factor()` explicita que el ráster resultante es categórico.

```{r}
rcl <- matrix(c(-Inf, 1.5, 1, 
                1.5, 2.5, 2, 
                2.5, Inf, 3), 
              ncol = 3, byrow = TRUE) 

reclasificado <- classify(r_pred, rcl = rcl) %>% as.factor()
```

Vectorización y conversión final: `as.polygons()` con [dissolve = TRUE]{.func-tecnica} agrupa todas las celdas contiguas de la misma clase y las fusiona en un único polígono por zona, eliminando los bordes internos de la grilla. El resultado es un [SpatVector]{.func-tecnica} con tres campos (uno por zona). La conversión a [sf]{.func-tecnica} y la asignación del CRS (EPSG:32720, WGS 84 / UTM zona 20S) devuelven el objeto estándar para realizar su gráfico:

```{r}
poligonos <- as.polygons(reclasificado, dissolve = TRUE)

poligonos_sf <- st_as_sf(poligonos) 

poligonos_sf <- st_set_crs(poligonos_sf, value = 32720) 
```

```{r}
datos_3Clases <- base_am %>%
```

Convertir el objeto [sf]{.func-tecnica} a [SpatVector]{.func-tecnica}.

Aunque [sf]{.func-tecnica} es excelente para manejar bases de datos geográficas, [terra]{.func-tecnica} es mucho más rápido para operaciones de rasterización.

```{r}
raster_base <- rast(puntos_terra, resolution = 10)
```

```{r}
raster_zonas <- rasterize(puntos_terra, raster_base, field = "X3clases")
```

```{r}
poligonos_terra <- as.polygons(raster_zonas, dissolve = TRUE)
```
