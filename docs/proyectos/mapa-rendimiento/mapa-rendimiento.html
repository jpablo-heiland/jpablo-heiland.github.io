1 Introducción

El presente documento describe el flujo de trabajo para la limpieza y visualización de una mapa de rendimiento mediante R. El propósito es eliminar las anomalías y errores de medición de los monitores de la cosechadora, transformando los datos “sucios” en información confiable para la toma de decisiones.

2 Librerías Necesarias

library(tidyverse)   
library(sf)   
library(knitr)
library(kableExtra)
library(ggspatial)

3 Lectura de Datos

Para la lectura de datos se usó read_csv() del paquete readr que forma parte del ecosistema tidyverse.

soja11 <- read_csv("data/Soja2011.csv")

Como se detalla en la Tabla 2 la clase del objeto es un Tibble. La función class() muestra toda la jerarquía de clases a la que pertenece ese objeto, ordenada desde la más específica hasta la más general (de izquierda a derecha), el significado de cada una es:

  • spec_tbl_df (Specification Tibble Data Frame): Indica que el objeto contiene una “especificación” oculta de las columnas. R guarda el registro de qué tipo de dato asignó automáticamente a cada columna.

  • tbl (Table): Es una clase genérica intermedia de la que heredan los tibbles.

  • data.frame: Es la estructura de datos base y más antigua de R para tablas. Al estar al final de la lista, significa que el tibble sigue siendo un data frame.

soja11 <- soja11 %>%

soja11 <- soja11 %>% 
  mutate(Elev11 = Elev11 * 0.3048)

7 Graficar datos sin limpiar

Antes de proceder a la limpieza del mapa de rendimiento se visualiza el mapa sin limpiar.

Como se mencionó en Tabla 2, la clase del set de datos inicial se estructuró como tibble. Esta carece de propiedades espaciales. Lo que impide graficarlo mediante ggplot.

Para resolver esto, se realizó una transformación hacia la clase Simple Features (SF) utilizando el paquete sf.

Un objeto de clase sf es, esencialmente, un data.frame que tiene una columna especial llamada geometry, la cual almacena la representación espacial (puntos, líneas o polígonos) que corresponde a cada fila.

Adicionalmente, se le asignó el Sistema de Referencia de Coordenadas (CRS) WGS 84 (EPSG:4326)

soja11_sf <- st_as_sf(soja11, coords = c("Lon", "Lat"), crs = 4326)

La visualización del rendimiento de soja se estructuró mediante una discretización en cuantiles (4), para ello, primero se deben crear estos cuantiles:

breaks <- quantile(soja11_sf$Rto11, probs = seq(0, 1, 0.25), na.rm = TRUE) 

soja11_sf <- soja11_sf %>% 
  mutate( 
    RTO11_q = cut(Rto11, breaks = breaks, include.lowest = TRUE, labels = paste0(round(breaks[-5],1), "–", round(breaks[-1],1))))

8 Limpiar datos

8.1 Eliminar de Inliers

Se usó la función filter() de la libreria dplyr (se encuentra dentro de tidyverse).

La función permite utilizar una o más condiciones. De esta forma el resultado obtenido serán aquellas observaciones donde se cumplan las restricciones indicadas.

  • Limpiar datos por Altura de plataforma:
soja11_sf <- soja11_sf %>% 
filter(Record01 == "Activ.")
  • Limpiar por Ancho de plataforma:
soja11_sf <- soja11_sf %>% 
filter(Width > 20)
  • Limpiar por Distancia:
soja11_sf <- soja11_sf %>% 
filter(Distance > 1.5)

8.2 Eliminar Outliers por Cuantiles

Se considerarán valores atípicos (outliers) aquellos valores que cumplan una de las siguientes condiciones:

  • El valor es menor que el primer cuartil menos 1,5 veces el rango intercuartílico = \(q < Q1 – 1,5 * IQR\)
  • El valor es mayor que el tercer cuartil más 1,5 veces el rango intercuartílico = \(q > Q3 + 1,5 * IQR\)

Calcular los cuantiles Q1 y Q3:

Q1 <- quantile(soja11_sf$Rto11, probs = 0.25, na.rm = TRUE)
Q3 <- quantile(soja11_sf$Rto11, probs = 0.75, na.rm = TRUE)

Calcular IQR:

IQR <- IQR(soja11_sf$Rto11)

Calcular los datos a eliminar:

eliminar1 = Q1 - (1.5 * IQR) 
eliminar2 = Q3 + (1.5 * IQR)

Eliminar Outliers:

soja11_sf <- soja11_sf %>% 
filter(Rto11 > eliminar1 & Rto11 < eliminar2)

9 Calcular el IP

El Índice de Productividad (IP) es una medida de rendimiento relativo. Se calcula dividiendo el rendimiento de cada punto individual por el rendimiento promedio de todo el lote.

Permite comparar zonas de un lote con otros lotes que tienen potenciales de rendimiento diferentes.

Interpretación:

  • IP > 1: La unidad rinde por encima del promedio. Un IP de 1.15 significa que rinde un 15% más que la media.
  • IP = 1: La unidad rinde lo mismo que el promedio.
  • IP < 1: La nidad rinde por debajo del promedio. Un IP de 0.85 significa que rinde un 15% menos que la media.
soja11_sf <- soja11_sf %>% 
mutate(IP = Rto11 / mean(Rto11))

```{r}
soja11_sf <- soja11_sf %>% 
filter(Rto11 > eliminar1 & Rto11 < eliminar2) 
```

# Calcular el IP

El **Índice de Productividad (IP)** es una medida de rendimiento relativo. Se calcula dividiendo el rendimiento de cada punto individual por el rendimiento promedio de todo el lote.

Permite comparar zonas de un lote con otros lotes que tienen potenciales de rendimiento diferentes.

**Interpretación**:

* IP > 1: La unidad rinde por encima del promedio. Un IP de 1.15 significa que rinde un 15% más que la media.
* IP = 1: La unidad rinde lo mismo que el promedio.
* IP < 1: La nidad rinde por debajo del promedio. Un IP de 0.85 significa que rinde un 15% menos que la media.

```{r}
soja11_sf <- soja11_sf %>% 
mutate(IP = Rto11 / mean(Rto11)) 
```
