1 Introducción

Este documento presenta el flujo de trabajo desarrollado en R para la limpieza, filtrado y visualización de un mapa de rendimiento de soja. El objetivo principal es detectar y remover errores y anomalías de medición generados por el monitor de la cosechadora. De este modo, se transforman los datos “sucios” en una capa de información confiable.

7 Espacialización y Proyección

7.1 Crear Objeto sf

Como se mencionó en la Tabla 2, el conjunto de datos inicial se estructuró como un tibble. Esta clase carece de propiedades espaciales. Para resolver esto, los datos se transformaron a la clase Simple Features (sf) mediante el paquete sf.

Un objeto de clase sf es un data.frame con una columna especial llamada geometry, que almacena la representación espacial (punto, línea o polígono) correspondiente a cada fila. Se convierte el tibble usando st_as_sf(), asignando las coordenadas en WGS 84 (EPSG:4326).

soja11_sf <- st_as_sf(
  soja11,
  coords = c("Lon", "Lat"),
  crs    = 4326
)

7.2 Reproyectar a UTM 20S

Se reproyecta a UTM Zona 20 Sur (EPSG:32720), sistema proyectado en metros adecuado para el centro-sur de Argentina.

soja11_sf <- st_transform(soja11_sf, crs = 32720)

9 Limpieza de Datos

El proceso combina dos estrategias complementarias: la eliminación de inliers (valores dentro del rango numérico pero inválidos por contexto operativo) y la eliminación de outliers estadísticos por el método IQR.

9.1 Paso 1 — Filtro por Estado de Registro (Record01)

El campo Record01 indica si la plataforma estaba activa en el momento del registro. Los puntos con estado desactivado corresponden a maniobras, cabeceras o traslados donde la cosechadora no estaba cosechando.

Primero se comprobaron los valores únicos de Record01:

soja11_sf %>%
  st_drop_geometry() %>%
  count(Record01, name = "n_registros") %>%
  mutate(porcentaje = percent(n_registros / sum(n_registros), accuracy = 0.1))
# A tibble: 2 × 3
  Record01    n_registros porcentaje
  <chr>             <int> <chr>     
1 Activ.            59626 95.1%     
2 Desactivado        3090 4.9%

Eliminar datos ‘Desactivado’:

soja11_sf  <- soja11_sf %>% filter(Record01 == "Activ.")
  Registros eliminados: 3.090 (4.9%)
  Registros restantes:  59.626

9.2 Paso 2 — Filtro por Ancho de Plataforma (Width)

En cabeceras o pasadas parciales, la plataforma opera a un ancho efectivo menor. Registros con anchos reducidos corresponden a zonas de solapamiento donde el rendimiento se distorsiona.

El umbral > 20 (pies) elimina pasadas donde la plataforma operaba a menos del ~30% de su ancho nominal.

soja11_sf <- soja11_sf %>% filter(Width > 20)
  Registros eliminados: 365 (0.6%)
  Registros restantes:  59.261

9.3 Paso 3 — Filtro por Distancia entre Puntos (Distance)

La distancia entre registros consecutivos es proporcional a la velocidad de avance. Valores muy bajos indican que la cosechadora estaba casi detenida, produciendo acumulaciones de puntos.

soja11_sf <- soja11_sf %>% filter(Distance > 1.5)
  Registros eliminados: 556 (0.9%)
  Registros restantes:  58.705

9.4 Paso 4 — Eliminación de Outliers por IQR

El método IQR detecta valores estadísticamente extremos. Se define el rango intercuartílico como la diferencia entre el tercer y primer cuartil, y se consideran outliers aquellos valores que caen afuera de las siguientes barreras:

\[ \text{Límite inferior} = Q_1 - 1.5 \times IQR \] \[ \text{Límite superior} = Q_3 + 1.5 \times IQR \] Calcular Cuantiles, IQR, Limite Inferior y Superior:

Q1  <- quantile(soja11_sf$Rto11, probs = 0.25, na.rm = TRUE)
Q2  <- quantile(soja11_sf$Rto11, probs = 0.50, na.rm = TRUE)
Q3  <- quantile(soja11_sf$Rto11, probs = 0.75, na.rm = TRUE)
IQR_val  <- IQR(soja11_sf$Rto11, na.rm = TRUE)

lim_inf <- Q1 - 1.5 * IQR_val
lim_sup <- Q3 + 1.5 * IQR_val

10 Calcular el IP

El Índice de Productividad (IP) es una métrica de rendimiento relativo que expresa el desempeño de cada punto del lote en relación al promedio general. Se calcula como:

\[ IP_i = \frac{Rto_i}{\overline{Rto}} \]

donde \(Rto_i\) es el rendimiento del punto \(i\) y \(\overline{Rto}\) es el rendimiento medio del lote.

Interpretación:

Valor IP Significado
IP > 1.15 Zona de alta productividad (≥15% sobre la media)
IP ≈ 1.00 Zona de productividad media
IP < 0.85 Zona de baja productividad (≥15% bajo la media)

El IP permite comparar zonas entre lotes con potenciales de rendimiento diferentes, ya que normaliza las diferencias absolutas de producción.

soja11_sf <- soja11_sf %>% 
mutate(IP = Rto11 / mean(Rto11))

## Paso 4 — Eliminación de Outliers por IQR

El método **IQR** detecta valores estadísticamente extremos. Se define el rango intercuartílico como la diferencia entre el tercer y primer cuartil, y se consideran outliers aquellos valores que caen afuera de las siguientes barreras:

$$
\text{Límite inferior} = Q_1 - 1.5 \times IQR
$$
$$
\text{Límite superior} = Q_3 + 1.5 \times IQR
$$
Calcular Cuantiles, IQR, Limite Inferior y Superior:

```{r}
#| label: iqr-calculo

Q1  <- quantile(soja11_sf$Rto11, probs = 0.25, na.rm = TRUE)
Q2  <- quantile(soja11_sf$Rto11, probs = 0.50, na.rm = TRUE)
Q3  <- quantile(soja11_sf$Rto11, probs = 0.75, na.rm = TRUE)
IQR_val  <- IQR(soja11_sf$Rto11, na.rm = TRUE)

lim_inf <- Q1 - 1.5 * IQR_val
lim_sup <- Q3 + 1.5 * IQR_val
```

# Calcular el IP

El **Índice de Productividad (IP)** es una métrica de rendimiento relativo que expresa el desempeño de cada punto del lote en relación al promedio general. Se calcula como:

$$
IP_i = \frac{Rto_i}{\overline{Rto}}
$$

donde $Rto_i$ es el rendimiento del punto $i$ y $\overline{Rto}$ es el rendimiento medio del lote.


Interpretación:

| Valor IP  | Significado                              |
|-----------|------------------------------------------|
| IP > 1.15 | Zona de alta productividad (≥15% sobre la media) |
| IP ≈ 1.00 | Zona de productividad media              |
| IP < 0.85 | Zona de baja productividad (≥15% bajo la media)  |

El IP permite comparar zonas entre lotes con potenciales de rendimiento diferentes, ya que normaliza las diferencias absolutas de producción.

```{r}
soja11_sf <- soja11_sf %>% 
mutate(IP = Rto11 / mean(Rto11)) 
```

# Exportación

Una vez completada la limpieza, se exporta el objeto [sf]{.func-tecnica} en dos formatos:

- **GeoPackage (`.gpkg`)**: recomendado para flujos de trabajo GIS. Conserva la geometría, el CRS y todos los atributos sin limitaciones de longitud de nombre de columna. Compatible con QGIS, ArcGIS y R/Python.
- **CSV (`.csv`)**: útil para importar los datos limpios en análisis estadísticos posteriores que no requieran información espacial (modelos, ANOVA, etc.).

```{r}
#| label: exportar
#| eval: false

dir.create("output", showWarnings = FALSE)

st_write(
  soja11_sf,
  "output/Soja2011_limpio.gpkg",
  layer         = "rendimiento",
  delete_layer  = TRUE
)

soja11_sf %>%
  st_drop_geometry() %>%
  write_csv("output/Soja2011_limpio.csv")
```
