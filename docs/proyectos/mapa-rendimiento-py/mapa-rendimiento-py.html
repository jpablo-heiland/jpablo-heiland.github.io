1 Introducción

El presente documento detalla el proceso de limpieza, filtrado y visualización de un mapa de rendimiento utilizando Python. El objetivo principal es eliminar las anomalías y errores de medición de los monitores de la cosechadora, transformando los datos “sucios” en información confiable para la toma de decisiones.

2 Configuración de Python

El procesamiento espacial se realizó con la librería GeoPandas en Python, ejecutada desde RStudio mediante el paquete reticulate. Dado que el entorno de Python utilizado corresponde a un ambiente Conda, fue necesario configurar manualmente la variable de entorno PROJ_DATA antes de la inicialización de pyproj, indicando la ruta local de la base de datos de proyecciones cartográficas PROJ. Esta configuración previa garantiza que el sistema de referencia de coordenadas EPSG:4326 (WGS84) sea reconocido correctamente al momento de construir el objeto GeoDataFrame.

library(reticulate)


proj_data <- "C:/Users/UsuarioX/miniconda3/envs/agro_science/Library/share/proj"
Sys.setenv(PROJ_DATA = proj_data)
Sys.setenv(PROJ_LIB  = proj_data)

use_python(
  "C:/Users/UsuarioX/miniconda3/envs/agro_science/python.exe",
  required = TRUE
)

9 Limpiar Datos

9.1 Limpiar Inliers

  • Limpiar datos por Altura de plataforma:

Comprobar los valores únicos de la columna Record01:

soja2011_gdf["Record01"].unique()
array([' Activ.', ' Desactivado'], dtype=object)

Eliminar datos ‘Desactivado’

soja2011_gdf_l = soja2011_gdf[soja2011_gdf["Record01"] == ' Activ.']
  • Limpiar por Ancho de Plataforma:
soja2011_gdf_l = soja2011_gdf_l[soja2011_gdf["Width"] > 20]
  • Limpiar por Distancia entre puntos:
soja2011_gdf_l = soja2011_gdf_l[soja2011_gdf["Distance"] > 1.5]

9.2 Limpiar Outliers por Rango Intercuartílico (IQR)

Se considerarán valores atípicos (outliers) aquellos valores que cumplan una de las siguientes condiciones:

  • El valor es menor que el primer cuartil menos 1,5 veces el rango intercuartílico = \(q < Q1 – 1,5 * IQR\)
  • El valor es mayor que el tercer cuartil más 1,5 veces el rango intercuartílico = \(q > Q3 + 1,5 * IQR\)

Calcular Cuantiles e IQR:

Q1 = soja2011_gdf_l["Rto11"].quantile(0.25) 
Q2 = soja2011_gdf_l["Rto11"].quantile(0.50)    
Q3 = soja2011_gdf_l["Rto11"].quantile(0.75) 
IQR = Q3 - Q1

Calcular los valores a eliminar:

eliminar1 = Q1 - (1.5 * IQR) 
eliminar2 = Q3 + (1.5 * IQR)

Eliminarlos:

soja2011_gdf_l = soja2011_gdf_l.loc[ 
  (soja2011_gdf_l["Rto11"] > eliminar1) & 
  (soja2011_gdf_l["Rto11"] < eliminar2)]

10 Calcular Indice de Productividad (IP)

El Índice de Productividad es una medida de rendimiento relativo. Se calcula dividiendo el rendimiento de cada punto individual por el rendimiento promedio de todo el lote.

Permite comparar zonas de un lote con otros lotes que tienen potenciales de rendimiento diferentes.

Interpretación:

  • IP > 1: La unidad rinde por encima del promedio. Un IP de 1.15 significa que rinde un 15% más que la media.
  • IP = 1: La unidad rinde lo mismo que el promedio.
  • IP < 1: La nidad rinde por debajo del promedio. Un IP de 0.85 significa que rinde un 15% menos que la media.
soja2011_gdf_l["IP"] = soja2011_gdf_l["Rto11"] / soja2011_gdf_l["Rto11"].mean()

* IP > 1: La unidad rinde por encima del promedio. Un IP de 1.15 significa que rinde un 15% más que la media.
* IP = 1: La unidad rinde lo mismo que el promedio.
* IP < 1: La nidad rinde por debajo del promedio. Un IP de 0.85 significa que rinde un 15% menos que la media.

```{python}
soja2011_gdf_l["IP"] = soja2011_gdf_l["Rto11"] / soja2011_gdf_l["Rto11"].mean()
```
