1 Introducción

El presente documento detalla el proceso de limpieza, filtrado y visualización de un mapa de rendimiento de soja utilizando Python. El objetivo principal es eliminar las anomalías y errores de medición de los monitores de la cosechadora, transformando los datos “sucios” en información confiable para la toma de decisiones.

2 Configuración de Python

El procesamiento espacial se realizó con la librería GeoPandas en Python, ejecutada desde RStudio mediante el paquete reticulate. Dado que el entorno de Python utilizado corresponde a un ambiente Conda, fue necesario configurar manualmente la variable de entorno PROJ_DATA antes de la inicialización de pyproj, indicando la ruta local de la base de datos de proyecciones cartográficas PROJ.

library(reticulate)


proj_data <- "C:/Users/UsuarioX/miniconda3/envs/agro_science/Library/share/proj"
Sys.setenv(PROJ_DATA = proj_data)
Sys.setenv(PROJ_LIB  = proj_data)

use_python(
  "C:/Users/UsuarioX/miniconda3/envs/agro_science/python.exe",
  required = TRUE
)

7 Proyección Espacial (UTM 20S)

7.1 Crear GeoDataFrame

Un GeoDataFrame es la estructura de datos de GeoPandas: una tabla (un pandas.DataFrame) que tiene una columna de geometría (un GeoSeries). Esa columna contiene objetos geométricos (por ejemplo Point, LineString, Polygon) que representan entidades espaciales.

soja2011_gdf = gpd.GeoDataFrame(
    soja2011,
    geometry=gpd.points_from_xy(soja2011["Lon"], soja2011["Lat"]),
    crs="EPSG:4326"
)

9 Limpieza de Datos

El proceso de limpieza combina dos estrategias: la eliminación de inliers (valores dentro del rango numérico pero inválidos por contexto operativo) y la eliminación de outliers estadísticos.

9.1 Paso 1 — Filtro por Estado de Registro (Record01)

El campo Record01 indica si la plataforma estaba activa en el momento del registro. Los puntos con estado “Desactivado” corresponden a momentos en que la cosechadora no estaba cosechando (maniobras, cabeceras, traslados).

Se comprobaron los valores únicos en Record01:

print(soja2011_gdf["Record01"].value_counts())
Record01
Activ.         59626
Desactivado     3090
Name: count, dtype: int64

Eliminar datos ‘Desactivado’:

soja2011_gdf_l = soja2011_gdf[soja2011_gdf["Record01"] == ' Activ.']
  Registros eliminados: 3,090 (4.9%)
  Registros restantes:  59,626

9.2 Paso 2 — Filtro por Ancho de Plataforma (Width)

Los registros con anchos muy reducidos corresponden a zonas de solapamiento donde el rendimiento se distorsiona.

El umbral de > 20 (en pies) elimina pasadas donde la plataforma estaba operando a menos del ~30% de su ancho nominal.

soja2011_gdf_l = soja2011_gdf_l[soja2011_gdf["Width"] > 20]
  Registros eliminados: 365 (0.6%)
  Registros restantes:  59,261

9.3 Paso 3 — Filtro por Distancia entre Puntos (Distance)

La distancia entre registros consecutivos es proporcional a la velocidad de avance. Valores muy bajos (< 1.5 ft) indican que la cosechadora estaba casi detenida, produciendo acumulaciones de puntos en el mismo lugar.

soja2011_gdf_l = soja2011_gdf_l[soja2011_gdf["Distance"] > 1.5]
  Registros eliminados: 556 (0.9%)
  Registros restantes:  58,705

9.4 Paso 4 — Eliminación de Outliers por Rango Intercuartílico (IQR)

El método IQR detecta valores estadísticamente extremos. Se define el rango intercuartílico como la diferencia entre el tercer y primer cuartil, y se consideran outliers aquellos valores que caen afuera de las siguientes barreras:

\[ \text{Límite inferior} = Q_1 - 1.5 \times IQR \] \[ \text{Límite superior} = Q_3 + 1.5 \times IQR \] Calcular Cuantiles, IQR y Limites:

Q1  = soja2011_gdf_l["Rto11"].quantile(0.25)
Q2  = soja2011_gdf_l["Rto11"].quantile(0.50)
Q3  = soja2011_gdf_l["Rto11"].quantile(0.75)
IQR = Q3 - Q1

lim_inf = Q1 - 1.5 * IQR
lim_sup = Q3 + 1.5 * IQR

Mostrar valores en una tabla:

Código
tabla_data = {
    "Métrica Estadística": [
        "Q1",
        "Q2",
        "Q3",
        "Rango Intercuartílico (IQR)",
        "Límite Inferior",
        "Límite Superior"
    ],
    "Valor (kg/ha)": [Q1, Q2, Q3, IQR, lim_inf, lim_sup]
}

df_iqr = pd.DataFrame(tabla_data)

df_iqr["Valor (kg/ha)"] = df_iqr["Valor (kg/ha)"].map("{:,.1f}".format)

df_iqr.style.hide(axis="index")

10 Calcular Indice de Productividad (IP)

El Índice de Productividad (IP) es una métrica de rendimiento relativo que expresa el desempeño de cada punto del lote en relación al promedio general. Se calcula como:

\[ IP_i = \frac{Rto_i}{\overline{Rto}} \]

donde \(Rto_i\) es el rendimiento del punto \(i\) y \(\overline{Rto}\) es el rendimiento medio del lote.

Interpretación:

Valor IP Significado
IP > 1.15 Zona de alta productividad (≥15% sobre la media)
IP ≈ 1.00 Zona de productividad media
IP < 0.85 Zona de baja productividad (≥15% bajo la media)

El IP permite comparar zonas entre lotes con potenciales de rendimiento diferentes, ya que normaliza las diferencias absolutas de producción.

soja2011_gdf_l["IP"] = soja2011_gdf_l["Rto11"] / soja2011_gdf_l["Rto11"].mean()
  IP mínimo:  0.534
  IP medio:   1.000
  IP máximo:  1.449

# Limpieza de Datos

El proceso de limpieza combina dos estrategias: la eliminación de **inliers** (valores dentro del rango numérico pero inválidos por contexto operativo) y la eliminación de **outliers** estadísticos.

## Paso 1 — Filtro por Estado de Registro ([Record01]{.func-tecnica})

El campo [Record01]{.func-tecnica} indica si la plataforma estaba activa en el momento del registro. Los puntos con estado "Desactivado" corresponden a momentos en que la cosechadora no estaba cosechando (maniobras, cabeceras, traslados).

Se comprobaron los valores únicos en [Record01]{.func-tecnica}:

```{python}
print(soja2011_gdf["Record01"].value_counts())
```

Eliminar datos 'Desactivado':

```{python}
soja2011_gdf_l = soja2011_gdf[soja2011_gdf["Record01"] == ' Activ.']
```

```{python}
#| echo: false

n_antes = len(soja2011_gdf)
```

```{python}
print(f"  Registros eliminados: {n_eliminados:,} ({n_eliminados/n_antes*100:.1f}%)")
print(f"  Registros restantes:  {len(soja2011_gdf_l):,}")
```

## Paso 4 — Eliminación de Outliers por Rango Intercuartílico (IQR)

El método IQR detecta valores estadísticamente extremos. Se define el rango intercuartílico como la diferencia entre el tercer y primer cuartil, y se consideran *outliers* aquellos valores que caen afuera de las siguientes barreras:

$$
\text{Límite inferior} = Q_1 - 1.5 \times IQR
$$
$$
\text{Límite superior} = Q_3 + 1.5 \times IQR
$$
Calcular Cuantiles, IQR y Limites:

```{python}
#| label: iqr-calculo

Q1  = soja2011_gdf_l["Rto11"].quantile(0.25)
Q2  = soja2011_gdf_l["Rto11"].quantile(0.50)
Q3  = soja2011_gdf_l["Rto11"].quantile(0.75)
IQR = Q3 - Q1

lim_inf = Q1 - 1.5 * IQR
lim_sup = Q3 + 1.5 * IQR
```

Mostrar valores en una tabla:

```{python}
#| code-fold: true
#| code-summary: "Código"

tabla_data = {
    "Métrica Estadística": [
        "Q1",
        "Q2",
        "Q3",
        "Rango Intercuartílico (IQR)",
        "Límite Inferior",
        "Límite Superior"
    ],
    "Valor (kg/ha)": [Q1, Q2, Q3, IQR, lim_inf, lim_sup]
}

df_iqr = pd.DataFrame(tabla_data)

df_iqr["Valor (kg/ha)"] = df_iqr["Valor (kg/ha)"].map("{:,.1f}".format)

df_iqr.style.hide(axis="index")
```

# Calcular Indice de Productividad (IP)

El **Índice de Productividad (IP)** es una métrica de rendimiento relativo que expresa el desempeño de cada punto del lote en relación al promedio general. Se calcula como:

$$
IP_i = \frac{Rto_i}{\overline{Rto}}
$$

donde $Rto_i$ es el rendimiento del punto $i$ y $\overline{Rto}$ es el rendimiento medio del lote.


Interpretación:

| Valor IP  | Significado                              |
|-----------|------------------------------------------|
| IP > 1.15 | Zona de alta productividad (≥15% sobre la media) |
| IP ≈ 1.00 | Zona de productividad media              |
| IP < 0.85 | Zona de baja productividad (≥15% bajo la media)  |

El IP permite comparar zonas entre lotes con potenciales de rendimiento diferentes, ya que normaliza las diferencias absolutas de producción.

```{python}
soja2011_gdf_l["IP"] = soja2011_gdf_l["Rto11"] / soja2011_gdf_l["Rto11"].mean()
```

```{python}
#| echo: false

print(f"  IP mínimo:  {soja2011_gdf_l['IP'].min():.3f}")
print(f"  IP medio:   {soja2011_gdf_l['IP'].mean():.3f}")
print(f"  IP máximo:  {soja2011_gdf_l['IP'].max():.3f}")
```
