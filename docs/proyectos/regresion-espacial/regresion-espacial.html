1 Introducción

El objetivo de este documento es evaluar el impacto agronómico y económico de la aplicación de densidad variable en maíz frente al menejo uniforme tradicional. Se procesan los datos provenientes de un ensayo de densidad de siembra de maíz atravesando zonas de manejo dentro de un lote de producción.

El punto de partida es la carga del ensayo y su exploración. Los datos de cosecha se agregan dentro de una cuadrícula de polígonos, sobre la cual se incorpora la capa de Zonas de Manejo del lote, asignando a cada celda la zona a la que pertenece.

Con el dataset consolidado se construyen las variables de contraste (dummy) que permiten estimar efectos diferenciados por zona.

Se calcula luego la matriz de vecinos bajo criterio Queen y se cuantifica la autocorrelación espacial del rendimiento mediante el Índice de Morán. A partir de este diagnóstico se comparan cuatro modelos: regresión clásica (OLS) y Modelo de Error Espacial (SEM), en versiones con y sin Zonas de Manejo.

De los coeficientes del modelo seleccionado se reconstruyen las funciones cuadráticas de respuesta del rendimiento a la densidad: una para el manejo uniforme y una por cada zona bajo manejo variable. Aplicando la primera derivada de cada función se calculan la Densidad Óptima Agronómica (DOAS) y la Densidad Óptima Económica (DOES), incorporando la relación de precios.

Finalmente, se realizan presupuestos parciales que cuantifican, en USD/ha y a escala de lote, el beneficio diferencial de aplicar densidad variable respecto del manejo uniforme.

5 Unir el Ensayo con la Cuadrícula

En primer lugar, se debe tener un identificador único de la cuadrícula.

La cuadrícula cargada como archivo shape tiene el campo OBS que funciona como identificador único:

colnames(cuadricula)
[1] "OBS"      "geometry"

En caso de crear la cuadrícula con sf se le debe asignar el identificador único (id), mediante la función mutate() en combinación con row_number().

cuadricula_sf <- cuadricula_sf %>%
  mutate(id_poligono = row_number())

Posteriormente, se ejecutó la unión de los datos del ensayo dentro de la estructura de la grilla. Este procedimiento se resolvió en cuatro etapas:

  • (st_join): Sirve para unir dos objetos espaciales (sf) basándose en su relación espacial. Se realizó una intersección espacial entre los polígonos de la cuadrícula y la capa de puntos del ensayo. Con el argumento left = FALSE se asegura que solo se conservaran aquellas celdas de la grilla que contuvieran datos reales de campo.

  • (group_by): Agrega una estructura de agrupamiento a los datos. En este caso, se agruparon los datos mediante la columna OBS.

  • (summarise): Esta función permite reducir múltiples filas de un dataset a valores resumidos, mediante la aplicación de funciones de agregación. Se estimó el valor medio de rendimiento y de densidad para cada polígono mediante la función mean().

  • Finalmente, se añadió el campo \(D^2\) (densidad al cuadrado), mediante mutate().

ensayo_D_sf <- cuadricula %>%
  st_join(ensayo, left = FALSE) %>% 
  group_by(OBS) %>%
  summarise(
    R_mean = mean(R, na.rm = TRUE),
    D_mean = mean(D, na.rm = TRUE),
    .groups = "drop"
  ) %>%
  mutate(
    D2 = D_mean^2
  )

6.1 Unir Zonas de Manejo al Ensayo

Se realizó una unión espacial para identificar a qué zona de manejo pertenece cada polígono de la cuadrícula del ensayo, es decir, se adicionó la Zona de Manejo a la capa Ensayo_D_sf. Este bloque se estructuró en:

  • Los polígonos correspondientes a cada celda de la cuadrícula (ensayo_D_sf) fueron transformados temporalmente en entidades de puntos mediante el cálculo de sus centroides mediante la función st_centroid().

  • Los centroides generados se intersectaron con la capa vectorial de las Zonas de Manejo (zm_sf) mediante una unión espacial (st_join()). Al operar con geometrías de puntos en lugar de polígonos, se fuerza al sistema a establecer una relación unívoca de tipo 1-a-1 (es decir, cada celda matemática es asignada estrictamente a una única zona de manejo). De esta manera, se evita la duplicación de registros en aquellas celdas ubicadas sobre los límites de transición.

  • Se eliminó el componente espacial, es decir, la columna geometry del objeto con st_drop_geometry(), para que el algoritmo funcione correctamente.

  • Finalmente, se filtraron los datos (select()) reteniendo exclusivamente las columnas adecuadas.

celdas_con_zona <- ensayo_D_sf %>%
  st_centroid() %>%                  
  st_join(zm_sf, left = TRUE) %>%    
  st_drop_geometry() %>%             
  select(OBS, Zonas, Sup)

Luego se realizó la consolidación de la base de datos final mediante los siguientes pasos:

  • Se integraron las zona de manejo y superficie (celdas_con_zona) a la cuadrícula (ensayo_D_sf), utilizando como llave primaria el identificador único de celda (OBS), mediante inner_join().

  • La columna correspondiente a las zonas de manejo (Zonas) fue transformada a un vector de tipo factor (as.factor). Esto le permite a los algoritmos de regresión interpretar la variable como niveles cualitativos discretos en lugar de cadenas de texto ordinarias, posibilitando la estimación diferenciada de interceptos y pendientes (efectos fijos) para cada zona de manejo.

ensayo_D_ZM_sf <- ensayo_D_sf %>%
  inner_join(celdas_con_zona, by = "OBS") %>% 
  mutate(
    Zonas = as.factor(Zonas)
  )

7.2 Crear Variables “Dummy”

Se crearon las variables “Dummy” para poder evaluar la significancia estadística de los corrimientos de los parámetros por zona de manejo.

ensayo_final_sf <- ensayo_D_ZM_sf %>%
  mutate(
    Z2 = case_when(
      Zonas == 1 ~ -1,
      Zonas == 2 ~ 1,
      TRUE ~ 0
    ),
    Z2D  = Z2 * D,
    Z2D2 = Z2 * D2,
    OBS = row_number()
  )

Las variables “dummy” se crearon en la columna Z2, que se generan mediante una condición lógica (case_when()). Esto analiza la columna Zonas de la celda y le asigna un valor numérico fijo de contraste:

  • Si la celda pertenece a la Zona 1, le da un valor de -1.

  • Si la celda pertenece a la Zona 2, le de un valor de un 1.

  • Si por algún motivo hay un dato raro o un borde sin zona, le pone 0.

Luego se creó Z2D de la siguiente manera \(\text{Z2D} = \text{Z2} \times \text{D}\)

Por último, se generó Z2D2: \(\text{Z2D2} = \text{Z2} \times \text{D2}\)

8 Matriz de Vecinos

Para el análisis de regresión espacial debemos, en primer lugar, crear la matriz de vecinos según la estructura Queen y luego, convertir esa lista en una matriz de pesos espaciales (W). Esto se realizó a de la siguiente forma:

  • Se utilizó la función poly2nb() para crear la matriz de vecinos, la cual identifica las celdas vecinas bajo el criterio de contigüidad tipo Reina (Queen) (queen = TRUE).

  • La lista de vecinos se transformó en una matriz de pesos espaciales mediante la función nb2listw(). Para ello se aplicó el estilo de ponderación “W”. Asimismo, se activó el argumento zero.policy = TRUE para prevenir la interrupción de los cálculos en caso de que existan celdas aisladas en los bordes del lote o zonas de descarte.

vecinos_queen <- poly2nb(ensayo_final_sf, queen = TRUE)

matriz_W_final <- nb2listw(vecinos_queen, style = "W", zero.policy = TRUE)

10 Realizar Regresiones Clásicas y Espaciales

Previo al análisis estadístico, se debe reescalar la densidad a “miles de semillas/ha” (dividiendo los valores por 1000). Esta modificación es un requisito técnico para evitar errores en el sistema: Al elevar la densidad original al cuadrado, se generan números muy grandes (doce mil millones). Trabajar en simultáneo con números tan gigantescos y valores tan chicos (como el \(0\) o \(1\) de las zonas de manejo) hace que el programa de R falle por un problema de desbordamiento numérico. El reescalado achica los números y estabiliza el cálculo.

ensayo_final_sf <- ensayo_final_sf %>%
  mutate(
    D_miles = D / 1000, 
    D2_miles = D_miles^2,
    Z2D_miles  = Z2 * D_miles,

14 Cálculo DOAS y DOES

A partir de las funciones cuadráticas estimadas para cada manejo y zona de manejo, se calcularon dos tipos de densidades óptimas mediante la aplicación de derivadas:

  • Densidad Óptima Agronómica (DOAS): Representa la cantidad de plantas necesaria para alcanzar el máximo rendimiento absoluto, sin considerar costos. Matemáticamente, se obtiene igualando la primera derivada de la función a cero (\(dY/dD = 0\)), lo que resulta en la fórmula:

\[\text{DOAS} = \frac{-b_1}{2 \cdot b_2}\]

  • Densidad Óptima Económica (DOES): Representa la densidad que maximiza el retorno económico. Se calcula igualando la primera derivada a la relación de precios (\(dY/dD = R_p\)), despejándose como:

\[\text{DOES} = \frac{R_p - b_1}{2 \cdot b_2}\]

14.1 Relación de Precios

Definir la relación de precios. El precio de maíz fue de 100 USD/tn (0.1 USD/kg), mientras que, el precio de la semilla fue de 180 USD/bolsa (0.0023 USD/semilla)

relacion_precios <-  2.3 / 0.1

Recordar que para realizar las regresiones se reescaló la densidad a miles de semillas/ha (dividiendo los valores por 1000). Al hacer esto el coeficiente lineal del modelo (\(\beta_1\)) deja de representar el ingreso por una sola semilla y pasa a representar el valor generado por cada bloque de 1.000 semillas extras. Se procedió a ajustar la escala del precio de la semilla a la misma unidad de medida del modelo (USD por cada 1.000 semillas). Así, el costo del insumo se fijó en \(2,3 \text{ USD/mil de semillas}\) (\(0,0023 \times 1.000\)), estableciendo una relación de precios final de \(23\) (\(2,3 \text{ USD/mil} / 0,1 \text{ USD/kg}\)).

# Introducción

El objetivo de este documento es evaluar el impacto agronómico y económico de la aplicación de densidad variable en maíz frente al menejo uniforme tradicional. Se procesan los datos provenientes de un ensayo de densidad de siembra de maíz atravesando zonas de manejo dentro de un lote de producción.

El punto de partida es la carga del ensayo y su exploración. Los datos de cosecha se agregan dentro de una cuadrícula de polígonos, sobre la cual se incorpora la capa de Zonas de Manejo del lote, asignando a cada celda la zona a la que pertenece.

Con el dataset consolidado se construyen las variables de contraste (*dummy*) que permiten estimar efectos diferenciados por zona.

Se calcula luego la matriz de vecinos bajo criterio *Queen* y se cuantifica la autocorrelación espacial del rendimiento mediante el Índice de Morán. A partir de este diagnóstico se comparan cuatro modelos: regresión clásica (OLS) y Modelo de Error Espacial (SEM), en versiones con y sin Zonas de Manejo.

De los coeficientes del modelo seleccionado se reconstruyen las funciones cuadráticas de respuesta del rendimiento a la densidad: una para el manejo uniforme y una por cada zona bajo manejo variable. Aplicando la primera derivada de cada función se calculan la **Densidad Óptima Agronómica (DOAS)** y la **Densidad Óptima Económica (DOES)**, incorporando la relación de precios.

Finalmente, se realizan presupuestos parciales que cuantifican, en USD/ha y a escala de lote, el beneficio diferencial de aplicar densidad variable respecto del manejo uniforme.

# Unir el Ensayo con la Cuadrícula

En primer lugar, se debe tener un identificador único de la cuadrícula.

En caso de crear la cuadrícula con [sf]{.func-tecnica} se le debe asignar el identificador único (id), mediante la función `mutate()` en combinación con `row_number()`.

```{r}

  mutate(id_poligono = row_number())
```

Posteriormente, se ejecutó la unión de los datos del ensayo dentro de la estructura de la grilla. Este procedimiento se resolvió en cuatro etapas:

-   (`st_join`): Sirve para unir dos objetos espaciales ([sf]{.func-tecnica}) basándose en su relación espacial. Se realizó una intersección espacial entre los polígonos de la cuadrícula y la capa de puntos del ensayo. Con el argumento [left = FALSE]{.func-tecnica} se asegura que solo se conservaran aquellas celdas de la grilla que contuvieran datos reales de campo.

-   (`group_by`): Agrega una estructura de agrupamiento a los datos. En este caso, se agruparon los datos mediante la columna [OBS]{.func-tecnica}.

-   (`summarise`): Esta función permite reducir múltiples filas de un dataset a valores resumidos, mediante la aplicación de funciones de agregación. Se estimó el valor medio de rendimiento y de densidad para cada polígono mediante la función `mean()`.

-   Finalmente, se añadió el campo $D^2$ (densidad al cuadrado), mediante `mutate()`.

```{r}
ensayo_D_sf <- cuadricula %>%
  st_join(ensayo, left = FALSE) %>% 
  group_by(OBS) %>%
  summarise(
    R_mean = mean(R, na.rm = TRUE),
    D_mean = mean(D, na.rm = TRUE),
    .groups = "drop"
  ) %>%
  mutate(
    D2 = D_mean^2
  )
```

## Unir Zonas de Manejo al Ensayo

Se realizó una unión espacial para identificar a qué zona de manejo pertenece cada polígono de la cuadrícula del ensayo, es decir, se adicionó la Zona de Manejo a la capa [Ensayo_D_sf.]{.func-tecnica} Este bloque se estructuró en:

-   Los polígonos correspondientes a cada celda de la cuadrícula ([ensayo_D_sf]{.func-tecnica}) fueron transformados temporalmente en entidades de puntos mediante el cálculo de sus centroides mediante la función `st_centroid()`.

-   Los centroides generados se intersectaron con la capa vectorial de las Zonas de Manejo ([zm_sf]{.func-tecnica}) mediante una unión espacial (`st_join()`). Al operar con geometrías de puntos en lugar de polígonos, se fuerza al sistema a establecer una relación unívoca de tipo 1-a-1 (es decir, cada celda matemática es asignada estrictamente a una única zona de manejo). De esta manera, se evita la duplicación de registros en aquellas celdas ubicadas sobre los límites de transición.

-   Se eliminó el componente espacial, es decir, la columna [geometry]{.func-tecnica} del objeto con `st_drop_geometry()`, para que el algoritmo funcione correctamente.

-   Finalmente, se filtraron los datos (`select()`) reteniendo exclusivamente las columnas adecuadas.

```{r}
celdas_con_zona <- ensayo_D_sf %>%
  st_centroid() %>%                  
  st_join(zm_sf, left = TRUE) %>%    
  st_drop_geometry() %>%             
  select(OBS, Zonas, Sup) 
```

Luego se realizó la consolidación de la base de datos final mediante los siguientes pasos:

-   Se integraron las zona de manejo y superficie ([celdas_con_zona]{.func-tecnica}) a la cuadrícula ([ensayo_D_sf]{.func-tecnica}), utilizando como llave primaria el identificador único de celda ([OBS]{.func-tecnica}), mediante `inner_join()`.

-   La columna correspondiente a las zonas de manejo (Zonas) fue transformada a un vector de tipo factor (`as.factor`). Esto le permite a los algoritmos de regresión interpretar la variable como niveles cualitativos discretos en lugar de cadenas de texto ordinarias, posibilitando la estimación diferenciada de interceptos y pendientes (efectos fijos) para cada zona de manejo.

```{r}
ensayo_D_ZM_sf <- ensayo_D_sf %>%
  inner_join(celdas_con_zona, by = "OBS") %>% 
  mutate(
    Zonas = as.factor(Zonas)
  )
```

## Crear Variables "Dummy"

Se crearon las variables “Dummy” para poder evaluar la significancia estadística de los corrimientos de los parámetros por zona de manejo.

```{r}
ensayo_final_sf <- ensayo_D_ZM_sf %>%
  mutate(
    Z2 = case_when(
      Zonas == 1 ~ -1,
      Zonas == 2 ~ 1,
      TRUE ~ 0
    ),

    Z2D2 = Z2 * D2,
    OBS = row_number()
  )
```

Las variables "dummy" se crearon en la columna Z2, que se generan mediante una condición lógica (`case_when()`). Esto analiza la columna Zonas de la celda y le asigna un valor numérico fijo de contraste:

-   Si la celda pertenece a la Zona 1, le da un valor de -1.

-   Si la celda pertenece a la Zona 2, le de un valor de un 1.

-   Si por algún motivo hay un dato raro o un borde sin zona, le pone 0.

Luego se creó Z2D de la siguiente manera $\text{Z2D} = \text{Z2} \times \text{D}$

# Matriz de Vecinos

Para el análisis de regresión espacial debemos, en primer lugar, crear la **matriz de vecinos** según la estructura Queen y luego, convertir esa lista en una **matriz de pesos espaciales (W)**. Esto se realizó a de la siguiente forma:

-   Se utilizó la función `poly2nb()` para crear la matriz de vecinos, la cual identifica las celdas vecinas bajo el criterio de contigüidad tipo Reina (Queen) ([queen = TRUE]{.func-tecnica}).

-   La lista de vecinos se transformó en una matriz de pesos espaciales mediante la función `nb2listw()`. Para ello se aplicó el estilo de ponderación "W". Asimismo, se activó el argumento [zero.policy = TRUE]{.func-tecnica} para prevenir la interrupción de los cálculos en caso de que existan celdas aisladas en los bordes del lote o zonas de descarte.

```{r}
vecinos_queen <- poly2nb(ensayo_final_sf, queen = TRUE)

matriz_W_final <- nb2listw(vecinos_queen, style = "W", zero.policy = TRUE)
```

# Realizar Regresiones Clásicas y Espaciales

Previo al análisis estadístico, se debe reescalar la densidad a "miles de semillas/ha" (dividiendo los valores por 1000). Esta modificación es un requisito técnico para evitar errores en el sistema: Al elevar la densidad original al cuadrado, se generan números muy grandes (doce mil millones). Trabajar en simultáneo con números tan gigantescos y valores tan chicos (como el $0$ o $1$ de las zonas de manejo) hace que el programa de R falle por un problema de desbordamiento numérico. El reescalado achica los números y estabiliza el cálculo.

```{r}
ensayo_final_sf <- ensayo_final_sf %>%
  mutate(
    D_miles = D / 1000, 
    D2_miles = D_miles^2,
    Z2D_miles  = Z2 * D_miles,
    Z2D2_miles = Z2 * D2_miles
```

# Cálculo DOAS y DOES

A partir de las funciones cuadráticas estimadas para cada manejo y zona de manejo, se calcularon dos tipos de densidades óptimas mediante la aplicación de derivadas:

-   **Densidad Óptima Agronómica (DOAS)**: Representa la cantidad de plantas necesaria para alcanzar el máximo rendimiento absoluto, sin considerar costos. Matemáticamente, se obtiene igualando la primera derivada de la función a cero ($dY/dD = 0$), lo que resulta en la fórmula:

$$\text{DOAS} = \frac{-b_1}{2 \cdot b_2}$$

-   **Densidad Óptima Económica (DOES)**: Representa la densidad que maximiza el retorno económico. Se calcula igualando la primera derivada a la relación de precios ($dY/dD = R_p$), despejándose como:

$$\text{DOES} = \frac{R_p - b_1}{2 \cdot b_2}$$

## Relación de Precios

Definir la relación de precios. El precio de maíz fue de 100 USD/tn (0.1 USD/kg), mientras que, el precio de la semilla fue de 180 USD/bolsa (0.0023 USD/semilla)

```{r}
relacion_precios <-  2.3 / 0.1
```

Recordar que para realizar las regresiones se reescaló la densidad a miles de semillas/ha (dividiendo los valores por 1000). Al hacer esto el coeficiente lineal del modelo ($\beta_1$) deja de representar el ingreso por una sola semilla y pasa a representar el valor generado por cada bloque de 1.000 semillas extras. Se procedió a ajustar la escala del precio de la semilla a la misma unidad de medida del modelo (USD por cada 1.000 semillas). Así, el costo del insumo se fijó en $2,3 \text{ USD/mil de semillas}$ ($0,0023 \times 1.000$), estableciendo una relación de precios final de $23$ ($2,3 \text{ USD/mil} / 0,1 \text{ USD/kg}$).
